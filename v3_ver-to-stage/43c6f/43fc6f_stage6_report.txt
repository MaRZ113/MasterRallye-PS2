Interpretation
- Stage6 is the first point where 43fc6f behaves like a usable extractor target instead of only a semantic-analysis domain.
- The prototype is intentionally conservative: it only emits manually validated clean media classes.
- Rejected / artifact-prone banks are excluded on purpose rather than being forced into output.

Next sensible step
- Stage7 should move from media-aware extraction to content-aware cataloguing:
  - assign stable human labels to the 3 clean challenge previews;
  - confirm whether the same preview bank is reused by multiple challenge records or whether there are hidden sub-variants inside the same banks;
  - expand the same extractor pattern to the next validated domain once enough manual truth exists.
